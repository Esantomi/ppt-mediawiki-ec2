미디어위키 설치

미디어위키 다운로드

  • 본 내용은 이전 강의를 통해 AWS EC2 인스턴스를 생성한 이후에 수강
  • AWS 홈페이지에 접속하여 로그인을 한 뒤, 아래 슬라이드를 참고하여 진행할 것!

복사+붙여넣기

1. cd ../../var/www/html/ 입력하여 경로 변경

2. sudo wget https://releases.wikimedia.org/
mediawiki/1.38/mediawiki-1.38.4.tar.gz 입력

3. sudo tar -xvzf mediawiki-1.38.4.tar.gz 입력

4. sudo mv mediawiki-1.38.4 wiki 입력

5. 퍼블릭 IP 주소로 접속했을 때 intl 설치가 필요하다면?

6. sudo apt-get install php-intl 입력하여 설치

7. Do you want to continue?가 뜨면 Y 입력

8. sudo service apache2 restart 입력

9. 주소 창에 퍼블릭IP주소/wiki/ 입력하여 접속

데이터베이스 및 사용자 생성

  • LAMP를 모두 갖추어 미디어위키 설치 준비는 완료함
    • Linux + Apache + MySQL, PHP
  • 하지만 데이터베이스 및 사용자 ID 생성은 직접!
    • 데이터를 저장할 공간(데이터베이스)과 그 공간을 사용할 사람의 ID를 만드는 과정
    • 아래 슬라이드 참고
  • 웹 SSH에 아래 명령어를 차례로 입력
    • 자세한 과정은 아래 슬라이드에서 상세히 진행

							sudo mysql -u root
							CREATE DATABASE wikidb;
							SHOW DATABASES;
							CREATE USER wiki@localhost IDENTIFIED BY 'e52fwUUfE9hyoi6r1';
							GRANT ALL PRIVILEGES ON wikidb.* TO wiki@localhost WITH GRANT OPTION;

1. sudo mysql -u root 입력하여 mysql 접속

2. mysql> 이 뜨면 무사히 접속한 것

3. CREATE DATABASE wikidb; 입력
: wikidb라는 이름의 데이터베이스 생성

4. SHOW DATABASES; 입력
: 데이터베이스 목록 보기

5. wikidb라는 이름의 데이터베이스 확인

6. CREATE USER wiki@localhost IDENTIFIED BY 'e52fwUUfE9hyoi6r1'; 입력하여 사용자 생성
(ID: wiki, PW: e52fwUUfE9hyoi6r1)

7. GRANT ALL PRIVILEGES ON wikidb.* TO wiki@localhost WITH GRANT OPTION; 입력
: 사용자(wiki)에게 데이터베이스에 대한 모든 권한 부여

7. quit 입력하여 mysql에서 빠져나오기

미디어위키 이미지 권한 부여

  • 위키를 이용하다 보면 이미지를 업로드를 해야 하는 경우가 빈번히 발생함
  • 위키 서버의 images 폴더에 이미지 파일이 저장됨
  • images 폴더의 권한을 변경해야 이미지 업로드 가능

1. sudo chmod 777 ../../var/www/html/wiki/images
: images 폴더의 권한을 777로 변경

  • 명령어 설명 : chmod
    • chmod 777 htdocs/wiki/images
      • chmod
        • 파일, 폴더의 권한을 변경(change mode)하는 명령어
      • 777
        • 사용자, 그룹, 기타에 대해 각각 7의 권한을 부여한다는 의미
        • 7 : 읽기(4) + 쓰기(2) + 실행(1) = 세 권한의 합
      • 즉, 누구든 위키에 이미지 파일을 올리거나 볼 수 있게 설정하는 과정!

2. sudo mv ../../var/www/html/wiki/images/.htaccess ../../var/www/html/wiki/images/.htaccess_disabled
: images 안 .htaccess 파일을 .htaccess_disabled로 변경

3. 입력 시 아무것도 안 뜬다면 잘 처리된 것

  • 명령어 설명 : mv
    • sudo mv ../../var/www/html/wiki/images/.htaccess ../../var/www/html/wiki/images/.htaccess_disabled
      • mv
        • 파일, 폴더를 이동(move)시키는 명령어
      • .htaccess
        • “hypertext access”의 약자
        • 폴더에 대한 접근을 설정하는 파일
      • 즉 .htaccess를 .htaccess_disabled로 바꾸는 것은 .htaccess 파일을 비활성화하는 것!

LocalSettings.php 서버 업로드

  • LocalSettings.php : 데이터베이스 정보를 포함, 위키와 관련된 대부분의 설정을 관리하는 파일
    • 미디어위키 설치 시 저절로 생성되나, 사용자가 직접 wiki 폴더 안에 넣어 줘야 함
  • 그런데 서버에 어떻게 파일을 올릴 수 있을까?🤔🤔
  • FTP/SFTP 프로그램인 FileZilla를 활용!
    • FTP(File Transfer Protocol) : 서버에 파일을 올리는 프로그램
    • SFTP(Secure FTP) : 암호를 통해 보다 안전하게 서버에 파일을 올리는 프로그램

1. FileZilla 설치 : 설치 방법은 이곳을 참고

2. FileZilla 실행

3. 파일 - 사이트 관리자 클릭

4. 새 사이트 클릭 후 이름을 ec2-wiki로 설정

5. 프로토콜, 호스트, 로그온 유형 키 파일로 입력 후
찾아보기로 전에 생성한 키 페어 파일(.pem) 입력 및 연결

6. 호스트는 인스턴스의 퍼블릭IP주소, 사용자는 인스턴스 사용자 이름을 입력한 것임

7. 알 수 없는 호스트키라는 창이 뜨면 '항상 이 호스트를 신뢰하고 이 키를 캐시에 등록(A)' 체크 후 확인 버튼 클릭

8. 서버에 성공적으로 연결된 상태

9. 리모트 사이트 아래 / 클릭

10. var 폴더를 찾아 더블클릭

11. www 폴더를 찾아 더블클릭

12. html 폴더 더블클릭

13. wiki 폴더 더블클릭

14. 다운받은 LocalSettings.php 파일 우클릭하여 업로드

미디어위키 추가 설정

미디어위키 로고 설정

  1. 로고로 사용할 135 x 135 픽셀 크기의 이미지 파일을 구한다. (저작권상 문제가 없어야 함)
  2. FileZilla로 wiki/resources/assets/ 폴더에 로고로 사용할 이미지 파일을 업로드한다.
  3. LocalSettings.php 파일을 우클릭한 뒤 메모장으로 연다. (기타 텍스트 에디터도 무관함)

미디어위키 로고 설정

  1. 38번째 줄-45번째 줄까지 아래 내용을 확인할 수 있다.
  2. 위의 change-your-logo.svg 부분을 로고로 사용할 이미지 파일의 이름으로 변경한다.
  3. 변경한 LocalSettings.php 파일을 서버 내 wiki 폴더로 업로드한다.

미디어위키 스킨 설정

  1. https://www.mediawiki.org/wiki/Category:All_skins/ko로 들어가서 원하는 스킨을 다운로드한다.
  1. 다운로드한 스킨의 압축을 풀고 wiki 폴더 내 skins 폴더에 다운로드한 스킨 폴더를 넣는다.
  2. 마찬가지로 LocalSettings.php 파일을 열면 128번째 줄-137번째 줄까지 아래 내용을 확인할 수 있다.
  1. LocalSettings.php에 wfLoadSkin('스킨 폴더명')을 추가함으로써 스킨을 설치하고, wgDefaultSkin의 값을 vector 대신 적용하고자 하는 스킨 폴더명으로 수정한다.
  2. 변경한 LocalSettings.php 파일을 서버 내 wiki 폴더로 업로드한다.

미디어위키 확장 기능 설치

  1. https://www.mediawiki.org/wiki/Category:
    Extensions_by_category/ko로 들어가서 원하는 확장 기능(extension)을 다운로드한다.
  1. 다운로드한 extension의 압축을 풀고 wiki 폴더 내 extensions 폴더에 다운로드한 extension 폴더를 넣는다.
  2. 마찬가지로 LocalSettings.php 파일을 열면 140번째 줄-173번째 줄까지 아래 내용을 확인할 수 있다.
  1. LocalSettings.php에 wfLoadSkin('extension 폴더명')을 추가함으로써 확장 기능을 설치하면 확장 기능이 자동으로 적용된다.
  2. 변경한 LocalSettings.php 파일을 서버 내 wiki 폴더로 업로드한다.

LocalSettings.php 설정

  • 아래 코드를 LocalSettings.php 파일 최하단에 삽입
    • 외부 이미지 허용하기
      • $wgAllowExternalImages = true;
      • 이미지 파일을 위키에 업로드하지 않아도 외부 이미지를 img 태그를 통해 가져올 수 있게 하는 설정
    • 새 창으로 띄우기
      • $wgExternalLinkTarget = '_blank';
      • 위키 내 링크를 클릭할 경우 새로운 창에서 해당 링크를 열게 해 주는 설정
  • 아래 코드를 LocalSettings.php 파일 최하단에 삽입
    • 권한이 없는 사용자의 문서 편집 막기
      • $wgGroupPermissions['*']['edit'] = false;
      • 다른 사람의 문서 수정 권한을 제거하여 혼자 글을 쓸 수 있는 위키로 변경
    • 미디어위키 가입을 차단하기
      • $wgGroupPermissions['*']['createaccount'] = false;
      • 다른 사람이 미디어위키에 가입하지 못하도록 변경
